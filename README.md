# Onboarding Redesign – Internship Task

## Overview
This project is a redesigned onboarding experience built using **Flutter**, focused on clarity, usability, and reducing user friction.

The goal was to help users quickly understand the value of the app and reach the main experience without feeling overwhelmed.

---

## Design Goals
- Keep onboarding short and focused
- Communicate benefits instead of long feature lists
- Reduce cognitive load
- Delay permissions and complex inputs
- Improve onboarding completion rate

---

## Onboarding Flow
1. **Welcome Screen** – Clear value proposition
2. **Feature Highlights** – Key benefits in simple language
3. **Role Selection** – Personalization without friction
4. **Completion Screen** – Clear transition to main app

---

## UX Decisions
- Limited onboarding to 4 screens to avoid drop-off
- Used icons and minimal text for faster comprehension
- Single primary action per screen
- Role selection improves relevance without forcing sign-up

---

## Tech Stack
- Flutter
- Material UI
- PageView for swipe-based onboarding
